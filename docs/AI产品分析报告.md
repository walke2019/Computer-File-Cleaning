## 1. 执行摘要

### 产品概述
AI 文件清理工具是一款基于人工智能的本地文件管理与清理软件，专为 Windows 平台设计。区别于传统清理工具的规则式清理，本产品采用 AI 驱动的智能分析，能够理解文件上下文、预测文件重要性，并提供个性化的清理建议。

### 核心价值主张
> **"不只是清理，更是智能整理"** - 100% 本地 AI 处理，保护隐私的同时释放存储空间

### 关键差异化优势
- 🧠 **语义理解**: 基于文件内容而非仅文件名进行智能分类
- 🔒 **隐私优先**: 100% 本地处理，无云端上传
- 🎯 **智能推荐**: AI 评估文件重要性，避免误删
- 💬 **自然语言**: 支持自然语言命令（如"删除去年的发票PDF"）

## 2. 市场分析

### 2.1 市场规模
| 指标 | 数据 |
|------|------|
| 全球市场规模 | $500M - $750M USD/年 |
| 年复合增长率 | 4-6% CAGR |
| 主要增长区域 | 亚太、北美 |

### 2.2 市场驱动因素
1. **数据爆炸**: 用户产生的数据量指数级增长（照片、视频、文档）
2. **SSD 成本**: 高端设备存储空间有限且昂贵，空间成为稀缺资源
3. **隐私意识**: 用户对追踪 Cookie、浏览器指纹和元数据越来越敏感

### 2.3 市场挑战
- **"蛇油"印象**: 市场充斥着夸大性能提升的广告软件
- **操作系统集成**: Windows (Storage Sense) 和 macOS 正在内置基础清理功能
- **安全顾虑**: 用户对授予第三方应用完整磁盘访问权限持谨慎态度

### 2.4 市场机会
| 机会领域 | 描述 |
|----------|------|
| 上下文智能 | 现有工具使用死板规则，缺乏对文件上下文的理解 |
| 智能删除 | 无工具能有效识别"你真的需要50张相似的日落照片吗？" |
| 云存储清理 | 大多数工具专注本地磁盘，很少有效管理云存储 |

## 4. 目标用户

### 4.1 用户画像

#### 画像 1: 创意工作者 / 内容创作者 💰💰💰
- **特征**: 设计师、摄影师、视频剪辑师
- **痛点**: 渲染时"磁盘已满"错误；成千上万未整理的素材、重复项和项目备份
- **需求**: 可视化空间占用、找到遗忘的大文件、安全清理不破坏项目链接
- **付费意愿**: 高

#### 画像 2: 数字囤积者 💰💰
- **特征**: 普通办公人员，多年积累大量文件
- **痛点**: 被杂乱淹没；害怕删除重要内容导致根本不敢清理
- **需求**: 安心感、"安全"清理、分类整理（如"这些是3年前下载从未打开的表情包"）
- **付费意愿**: 中

#### 画像 3: 隐私倡导者 💰💰
- **特征**: 技术敏感用户，关注数据安全
- **痛点**: 追踪 Cookie、浏览器历史、文件元数据
- **需求**: 安全删除（粉碎）、元数据清理、离线处理
- **付费意愿**: 中

#### 画像 4: 休闲用户 / 游戏玩家 💰
- **特征**: 普通消费者，非技术背景
- **痛点**: 电脑运行慢、游戏卡顿
- **需求**: "一键修复"、设置后忘记
- **付费意愿**: 低

## 5. 产品愿景与定位

### 5.1 愿景声明
> 成为用户信赖的"数字生活整理助手"，通过 AI 技术让文件管理变得简单、安全、智能。

### 5.2 使命
- 帮助用户从数字杂乱中解放
- 保护用户隐私和数据安全
- 让 AI 技术普惠每个人

### 5.3 核心差异化

| 传统清理工具 | 我们的 AI 工具 |
|--------------|----------------|
| 规则式清理（删除所有 .tmp 文件） | 上下文理解（知道哪个 .tmp 是正在编辑的文档） |
| 仅字节级重复检测 | 语义相似性（50张相似日落照片分组） |
| 被动工具 | 主动助手（"你有6个月没打开这些文件了"） |
| 黑箱操作 | 透明解释（"为什么 AI 选择了这个文件？"） |

### 6.2 技术栈选型

| 层级 | 技术选择 | 理由 |
|------|----------|------|
| **框架** | Electron + React | 跨平台能力、丰富生态、快速开发 |
| **UI** | Tailwind CSS + shadcn/ui | 现代化设计、高度可定制 |
| **核心逻辑** | TypeScript + Node.js | 类型安全、与 Electron 无缝集成 |
| **高性能模块** | Rust (via napi-rs) | 文件扫描、哈希计算等性能关键路径 |
| **AI 推理** | ONNX Runtime | 高效本地推理、跨平台 |
| **本地 LLM** | Ollama API | 支持多种开源模型、易于集成 |
| **数据库** | SQLite (better-sqlite3) | 轻量、无需安装、事务支持 |
| **打包** | electron-builder | 成熟的打包和自动更新方案 |

### 6.3 核心模块设计

#### 6.3.1 扫描引擎 (Scan Engine)
```typescript
interface ScanEngine {
  // 快速扫描（常用位置）
  quickScan(): Promise<ScanResult>;
  
  // 深度扫描（全盘）
  deepScan(paths: string[]): Promise<ScanResult>;
  
  // 增量扫描（仅变化文件）
  incrementalScan(): Promise<ScanResult>;
  
  // 实时监控
  watch(paths: string[]): FileWatcher;
}
```

#### 6.3.2 AI 分析引擎 (AI Analyzer)
```typescript
interface AIAnalyzer {
  // 文件分类
  categorize(file: FileInfo): Promise<Category>;
  
  // 重要性评分 (0-10, 10=安全删除)
  scoreRisk(file: FileInfo): Promise<number>;
  
  // 图像相似度分组
  groupSimilarImages(images: FileInfo[]): Promise<ImageGroup[]>;
  
  // 自然语言查询
  query(prompt: string): Promise<FileInfo[]>;
}
```

#### 6.3.3 清理引擎 (Clean Engine)
```typescript
interface CleanEngine {
  // 移动到隔离区（可恢复）
  quarantine(files: FileInfo[]): Promise<void>;
  
  // 永久删除
  delete(files: FileInfo[]): Promise<void>;
  
  // 安全粉碎（多次覆写）
  shred(files: FileInfo[]): Promise<void>;
  
  // 撤销上次操作
  undo(): Promise<void>;
}
```

| US-02 | 摄影师 | 作为创意工作者，我想要分组相似照片并只保留最佳的，避免存储被重复和连拍照片占满 | 1. AI 识别视觉相似性（不仅是完全重复）<br>2. UI 分组相似照片便于对比<br>3. 自动建议"最佳照片" |
| US-03 | 开发者 | 作为开发者，我想要找到并删除非活跃项目的 node_modules 和构建产物 | 1. 识别 .git、package.json 等开发文件夹<br>2. 按"最后修改日期"过滤<br>3. 允许批量删除特定文件夹 |
| US-04 | 办公人员 | 作为办公人员，我想要下载文件夹自动整理成"发票"、"安装包"、"个人"等子文件夹 | 1. AI 分析文件内容（不仅是扩展名）<br>2. 文件移动到分类子文件夹<br>3. 操作可撤销 |

### 7.2 功能需求列表

#### 核心扫描功能
| ID | 功能 | 优先级 | 描述 |
|----|------|--------|------|

| FR-SCAN-03 | 隐私清理 | P0 | 检测并清除浏览器痕迹（历史、Cookie）和应用最近文件列表 |
| FR-SCAN-04 | 大文件查找 | P1 | 可视化 >100MB 文件，按类型分类（视频、ISO、压缩包） |

| ID | 功能 | 优先级 | 描述 |
|----|------|--------|------|
| FR-AI-01 | 语义分类 | P1 | 使用本地 LLM/NLP 基于文件名语义和文本内容分类 |
| FR-AI-02 | 智能重要性评分 | P1 | 为文件分配"风险分数"(0-10)，10=安全，0=关键 |
| FR-AI-03 | 视觉相似度 | P2 | 使用本地 CV 模型查找重复或近似重复图像 |
| FR-AI-04 | 自然语言命令 | P2 | 支持查询如"删除去年的 PDF 发票" |

#### 清理优化功能
| ID | 功能 | 优先级 | 描述 |
|----|------|--------|------|
| FR-OPT-01 | 安全回收 | P0 | 默认操作为"移动到回收站"或"隔离文件夹"，非永久删除 |
| FR-OPT-02 | 撤销系统 | P0 | 24小时内可撤销上次批量操作 |
| FR-OPT-03 | 智能重命名 | P2 | 使用 AI 检测模式批量重命名（如 IMG_001.jpg → 东京之旅_01.jpg） |

#### 设置与偏好
